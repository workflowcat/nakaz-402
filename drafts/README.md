# drafts/

Робочі проєкти амендментів до Наказу МОУ № 402.

**Це НЕ офіційні проєкти нормативно-правових актів.** Це машино-читабельні
ескізи майбутніх правок — від внутрішніх юристів МОУ, ЦВЛК-експертів,
адвокаційних організацій і civic-tech контриб'юторів.

Кожен файл `*.md` — один проєкт. YAML-frontmatter описує операції
(`redaction`, `insert`, `repeal`, `restore`) над конкретними пунктами
(stable ID на кшталт `polozhennia.r1.gl1.p1.4`). Markdown-тіло — пояснення
для людей.

## Як подати свій проєкт

1. Створіть гілку: `draft-<рік>-<короткий-slug>`.
2. Додайте файл `drafts/<рік>-<NNN>-<slug>.md`.
3. Заповніть `status: draft` (потім можна підвищити до `review`, далі —
   `adopted` якщо МОУ фактично прийняв подібний наказ, або `rejected`).
4. Відкрийте PR. CI запустить лінтер: чи існують target-ID, чи немає
   термінологічного дрейфу, чи не конфліктує з іншими відкритими драфтами.
5. На сайті проєкт автоматично з'явиться на `/drafts/<id>/` із live-preview.

## Статуси

| Статус       | Значення |
|--------------|----------|
| `draft`      | Чернетка, обговорюється |
| `review`     | Готовий до огляду стейкхолдерами |
| `adopted`    | Подібні зміни прийняті офіційно (вказати № наказу-амендменту) |
| `rejected`   | Знятий з розгляду |
