# Додаток 1. Розклад хвороб

Затверджений Положенням про військово-лікарську експертизу
(пункт 1 [Наказу МОУ № 402](../../nakaz.md)).

> 🟡 **Stub.** У повному вигляді Розклад хвороб містить **~80—90 статей**,
> згрупованих за класами хвороб (інфекційні, новоутворення, хвороби крові,
> ендокринні, психічні, нервової системи, ока, вуха, кровообігу, дихання,
> травлення, шкіри, кістково-м'язові, сечостатеві, вроджені аномалії,
> наслідки травм, тощо). Тут наведено демо-структуру і одну заповнену статтю.

## Структура

```
01-rozklad-khvorob/
├── README.md            ← цей файл, опис підходу
├── meta.yaml            ← схема статті, перелік граф і категорій придатності
├── stattia-001.yaml     ← Туберкульоз органів дихання  (демо)
├── stattia-014.yaml     ← Реактивні психози і невротичні розлади  (демо)
└── stattia-042.yaml     ← Артеріальна гіпертензія  (демо)
```

Значення категорій у демо-статтях **ілюстративні**, не звіряні з чинною
редакцією. Перед використанням у будь-яких рішеннях звіряйте з офіційним
текстом на rada.gov.ua.

## Як читати статтю

Кожна стаття — окремий файл `stattia-NNN.yaml`. У файлі:

- **`stattia`** — номер статті;
- **`nazva`** — назва (заголовок групи хвороб);
- **`punkty`** — список підпунктів (зазвичай позначаються літерами «а», «б»,
  «в» — за ступенем важкості);
- **`grafy`** — категорії придатності за різними графами:
  - **Графа I** — призовники;
  - **Графа II** — військовослужбовці строкової служби, курсанти ВВНЗ;
  - **Графа III** — військовослужбовці контрактної служби, офіцери;
  - **Графа IV** — особи, які приймаються на службу за контрактом, та військовозобов'язані.

Кожна категорія в графі — це один із кодів **А / Б / В / Г / Д** з підваріантами
(див. [`meta/glossary.md`](../../meta/glossary.md)).

## Звідки взагалі ідея даних, а не тексту

Сьогодні у Положенні стаття пишеться приблизно так:

> «Стаття 1. Туберкульоз органів дихання.
> а) активний прогресуючий туберкульоз — **Графа I:** Д; **Графа II:** Д;
> **Графа III:** Д; **Графа IV:** Д.
> б) активний з обмеженою активністю — **Графа I:** В; **Графа II:** В;
> **Графа III:** Б-4; **Графа IV:** В.
> ...»

Це по суті — таблиця у форматі прози. Якщо записати її як таблицю, то:

- редагування пункту «б, графа III» = одна правка в YAML;
- лінтер скаже, якщо забули заповнити графу;
- можна автоматично згенерувати «що зміниться, якщо в статті 1 пункт б графа III
  поміняти Б-4 на В».

Така структура — фундамент для будь-якого «калькулятора придатності»: на вхід
діагноз, на вихід — категорія за всіма графами.

## Демо-калькулятор

Скрипт [`scripts/fitness_calculator.py`](../../scripts/fitness_calculator.py)
читає ці YAML-файли і відповідає на типові питання:

```bash
# Перелічити всі статті, що є в репо
$ python3 scripts/fitness_calculator.py --list
Знайдено 3 статей у dodatky/01-rozklad-khvorob/:
    1  Туберкульоз органів дихання
   14  Реактивні психози і невротичні розлади
   42  Хвороби, які характеризуються підвищеним кров'яним тиском (артеріальна гіпертензія)

# Деталі однієї статті
$ python3 scripts/fitness_calculator.py --stattia 42
Стаття 42: ... артеріальна гіпертензія ...
  Підпункти:
    а) зі значним порушенням функції... — графи: I:Д, II:Д, III:Д, IV:Д
    б) з помірним порушенням...        — графи: I:В, II:В, III:В, IV:В
    в) з незначним порушенням...       — графи: I:Б-4, II:Б-4, III:Б-3, IV:Б
    г) нейроциркуляторна дистонія...   — графи: I:Б-3, II:Б-3, III:А, IV:Б

# Прицільно за статтею + підпунктом + графою
$ python3 scripts/fitness_calculator.py --stattia 1 --punkt б --graf III
Стаття 1: Туберкульоз органів дихання  [active]
  Підпункт «б»: активний обмежений, із сповільненою динамікою...
  Графа III (Військовослужбовці контрактної служби, офіцери): Б-4 — Придатний до служби з обмеженнями за видом

# JSON для інтеграції в зовнішні сервіси
$ python3 scripts/fitness_calculator.py --stattia 42 --json
```

Це і є **результат «конфіг як дані»**: з простого YAML без жодних додаткових
зусиль виходить програмний доступ. Реальний інструмент може бути:

- веб-сторінкою з пошуком за діагнозом → стаття → категорія;
- модулем у HR-системі ЗСУ для попередньої оцінки;
- API для інтеграції з електронною ВЛК-системою.

Якщо завтра наказом-зміною оновлено категорію в одному підпункті — це **одна
правка в YAML**, і калькулятор автоматично відповідає інакше.
