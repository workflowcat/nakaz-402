# campaigns/ — advocacy-кампанії

Тут зберігаються координовані «кампанії» — набори пов'язаних драфтів,
які підштовхує одна організація або коаліція. На відміну від окремого
драфту (одна правка), кампанія = декілька правок з спільною метою,
спільним меседжингом і спільною стратегією просування.

**Це не офіційні документи.** Це робочий простір для адвокаційних
організацій (Veteran Hub, Right to Protection, БМА), юристів МОУ,
ЦВЛК-експертів і громадянських ініціатив — щоб координувати, що саме
вони лобіюють і чому.

## Структура файла

```yaml
---
id: campaign-2026-001
title: "Захист персональних даних у Трембіті"
owner: workflowcat              # Хто веде кампанію
created_at: 2026-05-14
status: active                   # active | concluded | paused | dropped
goal: >-
  Що в підсумку має зробити МОУ
target_audience:
  - "Юридичний департамент МОУ"
  - "Уповноважений ВР з прав людини"
member_drafts:
  - draft-2026-001               # Посилання на драфти у drafts/
  - draft-2026-002
key_messages:                    # Спрощені тези для пресс-релізів
  - "Чинна редакція дозволяє безконтрольний обмін..."
references:
  - title: "GDPR Art. 30"
    url: https://...
---

Розгорнутий бриф у markdown — обґрунтування, метрики успіху,
наступні кроки.
```

## Як подати кампанію

1. Створіть гілку `campaign-<рік>-<slug>`.
2. Додайте `campaigns/<рік>-<NNN>-<slug>.md`.
3. Перелічіть `member_drafts` за їхніми id (вони повинні існувати в `drafts/`).
4. Опишіть `goal` і `target_audience`.
5. Відкрийте PR. На сайті з'явиться `/campaigns/<slug>/` зі зведеним
   lint-статусом, картою впливу на главам Положення і списком кроків.
