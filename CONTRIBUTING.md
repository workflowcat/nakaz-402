Цей репозиторій тримає **інженерну реплікацію офіційного НПА** у машино-читабельному
вигляді. Будь-які зміни мають бути верифіковні **за офіційним джерелом**
([zakon.rada.gov.ua/laws/show/z1109-08](https://zakon.rada.gov.ua/laws/show/z1109-08)).

Робота розкладається на три типи:

| Тип контрибуції                | Тригер                                     | Шаблон                                |
| ------------------------------ | ------------------------------------------ | ------------------------------------- |
| **Амендмент** (наказ-зміна)    | Вийшов новий наказ Міноборони, що змінює 402 | [Issue → New amendment](.github/ISSUE_TEMPLATE/new-amendment.yml) |
| **Контент** (заповнення заглушок) | У файлі є `stub: true` і ви маєте час перенести текст | [Issue → Content fix](.github/ISSUE_TEMPLATE/content-fix.yml) |
| **Інфраструктура**             | Лінтер, скрипти, тема, GitHub Action       | звичайний PR                          |

## 1. Базове налаштування

Один раз:

```bash
git clone <repo-url>
cd nakaz-402

# Залежності для лінтера / генераторів
pip install pyyaml

# Залежності для локального jekyll serve (опціонально)
bundle install
```

## 2. Внесення амендменту (наказ-зміна) — головний кейс

Це той сценарій, заради якого все і робилось. Покроково:

### 2.1. Створіть гілку

Конвенція: `amendment/<номер_наказу>-<дата_у_iso>`

```bash
git checkout -b amendment/153-2025-02-27
```

### 2.2. Внесіть текстові правки

Знайдіть зачеплені файли в `polozhennia/` або `dodatky/`. Для кожного:

1. **Оновіть текст** відповідно до офіційного джерела.
2. **Додайте запис у `amended_by`** у frontmatter:

```yaml
amended_by:
  # ... попередні
  - { date: 2025-02-27, order: z0329-25, op: insert, scope: "новий п. 1.4" }
```

Допустимі значення `op`:
- `insert` — додано новий абзац/пункт;
- `edit` — дрібна правка (слово, формулювання, посилання);
- `redaction` — повністю нова редакція пункту;
- `repeal` — пункт втратив чинність (текст залишаємо, ставимо `status: repealed`);
- `restore` — рідко, але можливо: відновлення раніше скасованого.

3. **Збережіть в тексті** маркер у форматі rada.gov.ua (фігурні дужки) — це
   допоможе очима звіряти з офіційним джерелом:

```markdown
> *{Главу 1 розділу I доповнено новим пунктом 1.4 згідно з Наказом № 153
> від 27.02.2025 ([z0329-25](https://zakon.rada.gov.ua/laws/show/z0329-25#n22))}*
```

### 2.3. Оновіть `meta/amendments.yaml`

Додайте запис:

```yaml
- order: "153"
  signed_at: 2025-02-27
  registration: z0329-25
  summary: >-
    Доповнено главу 1 розділу I новим пунктом 1.4
    (електронна інформаційна взаємодія через «Трембіту»).
  affects:
    - id: polozhennia.r1.gl1.p1.4
      op: insert
```

### 2.4. Перегенеруйте CHANGELOG

```bash
make changelog
# або вручну
python3 scripts/changelog_from_amendments.py --write
```

### 2.5. Прогоніть лінтер локально

```bash
make lint
```

Має бути «0 помилок».

### 2.6. Створіть PR

Шаблон підставиться автоматично:

```bash
git add .
git commit -m "amendment: наказ № 153 від 27.02.2025 — новий п. 1.4 (z0329-25)"
git push -u origin amendment/153-2025-02-27
gh pr create
```

### 2.7. Тегування редакції після мержу

Після того як PR мержать у `main`, мейнтейнер тегує:

```bash
git tag redaction/2025-02-27 -m "Редакція станом на 2025-02-27 (наказ № 153)"
git push --tags
```

Це створює стабільну точку «офіційна редакція станом на дату X».

## 3. Заповнення заглушки (контент)

У багатьох файлах сьогодні `stub: true`. Якщо хочете перенести текст з
офіційного джерела:

1. Зніміть `stub: true` з frontmatter.
2. Перенесіть текст пункт за пунктом, **зберігаючи маркери змін** у фігурних дужках.
3. Для кожного пункту, де ви бачите маркер «{Із змінами, внесеними згідно з
   Наказом № ...}», додайте відповідний запис у `amended_by`.
4. Якщо помітили розбіжності між текстом тут і офіційним джерелом — створіть
   окремий issue (це може бути слід амендменту, який пропустили).

Це довший і кропіткіший трип, ніж амендмент. Перевага — кожна нова заповнена
глава одразу стає доступною на сайті з повним пошуком.

## 4. Інфраструктурні PR

Звичайний PR. Якщо змінюєте:

- **frontmatter-схему** (`meta/schema.md`) — оновіть і `scripts/lint_frontmatter.py`;
- **генератори** (`scripts/`) — переконайтесь, що CI всі перевірки проходить;
- **тему сайту** — перевірте локально (`bundle exec jekyll serve`);
- **`_config.yml`** — пам'ятайте, що `baseurl` може бути `/<repo>` на pages.

---

## 5. Чого НЕ робимо

- **Не пишемо текст НПА від себе.** Усе має бути з офіційного джерела.
- **Не видаляємо скасовані пункти.** Ставимо `status: repealed`, лишаємо текст.
- **Не амендимо `amendments.yaml` без оновлення CHANGELOG** (CI попередить).
- **Не міняємо `id` існуючих пунктів.** Вони стабільні навіки. Якщо
  нумерація змінилась — це окрема історія («restructure»), погоджується
  з мейнтейнерами.

## 6. Швидкий чек-ліст для амендменту

```
[ ] Гілка amendment/<order>-<date>
[ ] У зачеплених .md — оновлено текст
[ ] У frontmatter кожного зачепленого .md — додано запис у amended_by
[ ] У тексті — маркер у фігурних дужках з посиланням на наказ-зміну
[ ] У meta/amendments.yaml — новий запис
[ ] make changelog (або scripts/changelog_from_amendments.py --write)
[ ] make lint — 0 помилок
[ ] PR створено, шаблон заповнено
[ ] Звірено з офіційним текстом на rada.gov.ua
```
