# Зразки згенерованих проєктів наказів

Файли в цій теці згенеровано скриптом
[`scripts/draft_amendment.py`](https://github.com/workflowcat/nakaz-402/blob/main/scripts/draft_amendment.py)
з реальних записів `meta/amendments.yaml`. Це **демонстрація**: показує, як
з машино-читабельних даних вийде готовий шаблон у форматі МОУ для підпису.

## Що всередині кожного `.docx`

1. **Шапка** — герб України (placeholder), «МІНІСТЕРСТВО ОБОРОНИ УКРАЇНИ», «НАКАЗ»
2. **Реквізити** — дата, місце, номер у трьох колонках
3. **Заголовок** — «Про внесення змін до Положення про…»
4. **Преамбула** — «Відповідно до… з метою… НАКАЗУЮ:»
5. **Тіло наказу** — для кожного зачепленого `id`:
   - `insert` → «Доповнити … новим пунктом такого змісту: …»
   - `redaction` → «Викласти … у такій редакції: …»
   - `edit` → «У … слова «…» замінити словами «…»»
   - `repeal` → «Виключити …»
   - `restore` → «Відновити дію …»
6. **Пункт про контроль виконання**
7. **Прикінцеві положення**
8. **Підпис** — «Міністр оборони України» + placeholder
9. **ДОДАТОК — порівняльна таблиця** (на окремій сторінці):
   - три колонки: `id` пункту · поточна редакція · запропонована редакція
   - поточна редакція автоматично витягується з YAML-файлів положення
   - для `repeal` — поточний текст з ефектом strikethrough
10. **Довідкові відомості** — таблиця метаданих: планова реєстрація,
    джерело-наказ, кількість зачеплених пунктів, типи операцій, резюме,
    обґрунтування

## Чи це готова форма для підпису?

Ні. Це **робочий draft з 80% контексту** — готова структура, заповнені
метадані, витягнутий поточний текст у порівняльну таблицю, правильні
формулювання типу операції. Залишилось:

- вписати точний новий текст у поля `[ ТЕКСТ_… ]`;
- заповнити «Обґрунтування» (зазвичай йде в окрему пояснювальну записку);
- проставити підпис, дату.

Це економить ~5 годин роботи юриста, який зазвичай **вручну** збирає
порівняльну таблицю, копіює поточний текст з консультанта-плюс або
rada.gov.ua, форматує реквізити.

## Як це згенерувати

```bash
# З уже наявного запису в amendments.yaml
python3 scripts/draft_amendment.py --from-history z0329-25

# З власного YAML-spec'а для майбутнього амендменту
cat > my-draft.yaml <<EOF
order: "999"
signed_at: 2026-06-01
registration: z9999-26
summary: "Доповнення Положення новим пунктом про …"
rationale: "Обґрунтування: …"
affects:
  - id: polozhennia.r1.gl1.p1.5
    op: insert
    new_text: "Тут новий текст…"
EOF
python3 scripts/draft_amendment.py --input my-draft.yaml

# Або через Makefile
make draft AMENDMENT=z0329-25
```

## Що в планах поверх

- **Пояснювальна записка** — окрема `scripts/draft_explanatory_note.py`.
- **Web-форма генерації** — на сайті, без CLI.
- **AI-помічник** — пропонує текст нового пункту і обґрунтування на основі
  поточної редакції і summary амендменту.
- **Двостороння інтеграція** — після підписання справжнього наказу
  автомат-парсер створює PR у репо з відповідним записом.
